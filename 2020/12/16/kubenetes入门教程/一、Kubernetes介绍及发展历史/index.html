一、Kubernetes介绍及发展历史

Posted by 佘黎明 on 2020-12-16

Kubernetes的名字来自希腊语，意思是“舵手” 或 “领航员”。K8s是将8个字母“ubernete”替换为“8”的缩写。

一、概述

是Google开源的容器集群管理系统，最初是内部Borg。是容器编排调度引擎。

为跨主机的容器化应用提供资源调度、服务发现、高可用管理、弹性伸缩功能。

它不仅仅是一个编排系统，而是提供一个规范描述集群的架构，定义服务的最终状态，k8s自动帮你将系统达到和维持这个状态。

同类产品

Docker Swarm 2019.07 阿里云不提供

kubernetes Google 10年前 borg Go重写

二、特性

  1. 自动化部署：yaml部署k8s，自动分配到node
  2. 系统自愈：node当机，k8s会重新将pod调用到可用节点
  3. 水平扩展：HPA周期调整RC的副本数量
  4. 服务发现和负载均衡：为每个容器分配IP，service代理
  5. 自动更新和回滚：不停应用

轻量级：资源消耗小

InitC:初始化容器，顾名思义容器启动的时候，会先启动可一个或多个容器，如果有多个，那么这几个Init Container按照定义的顺序依次执行，只有所有的Init Container执行完后，主容器才会启动。由于一个Pod里的存储卷是共享的，所以Init Container里产生的数据可以被主容器使用到。

安全

Kubernetes 作为一个分布式集群的管理工具，保证集群的安全性是其一个重要的任务。API Server 是集群内部各个组件通信的中介，也是外部控制的入口。所以 Kubernetes 的安全机制基本就是围绕保护 API Server 来设计的。Kubernetes 使用了认证（Authentication）、鉴权（Authorization）、准入控制（Admission Control）三步来保证API Server的安全
